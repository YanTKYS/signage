# iSignage

`isignage.html` だけで動作する、ローカル用の単一HTMLサイネージ作成・表示ツールです。PowerPointに近い感覚で、テキスト、画像、動画、フォルダ再生メディア、時計、図形を16:9キャンバスへ配置できます。

## 対応ブラウザ

- Microsoft Edge 最新版
- Google Chrome 最新版

File System Access API を利用するため、Firefox や Safari ではフォルダ選択機能が利用できない場合があります。

## 起動方法

1. `isignage.html` をローカル PC に置きます。
2. Edge または Chrome で直接開きます。
3. Web サーバー、npm、外部ライブラリ、CDN は不要です。

## 基本操作

- 上部の「テキスト」「画像」「動画」「メディア」「図形」「時計」から追加します。
- 上部の「? ヘルプ」から、基本操作、メディアフォルダ、シーン、保存、ショートカット、注意事項を同一HTML内のモーダルで確認できます。外部通信や別ページ表示は行いません。
- 初回起動時は簡単な案内を表示します。「使い方を見る」から詳細ヘルプを開けます。「次回から表示しない」を選ぶと、次回起動時は自動表示しません。
- オブジェクトはクリックで選択、ドラッグで移動できます。
- 選択枠の 8 つのハンドルでサイズ変更できます。画像と動画は既定で四隅ハンドルのリサイズ時に縦横比を維持し、`Shift` キーで一時的に維持/解除を反転できます。
- キャンバスの空白部分をドラッグすると範囲選択できます。`Ctrl` を押しながら範囲選択すると既存選択へ追加します。
- `Delete` 削除、`Ctrl+C` コピー、`Ctrl+V` 貼り付け、`Ctrl+D` 複製、`Ctrl+Z` Undo、`Ctrl+Y` Redo に対応しています。
- 右側のプロパティ欄から位置、サイズ、色、透明度、再生設定などを変更できます。不正な数値は安全な範囲へ補正されます。
- 利用できない配置ボタンやシーン操作ボタンは無効表示になります。

## シーン設定

左ペインの「シーン設定」から、現在のシーン名、背景色、表示秒数を編集できます。表示秒数は 1 秒以上の安全な値へ補正されます。シーン切替方法が「切り替えない」の場合も設定値は保持されます。

左シーン欄は「‹ シーン欄を閉じる」で折りたためます。折りたたみ中もキャンバス左端に「› シーン欄を開く」ボタンが残るため、そこから再表示できます。開閉状態はUI設定として自動保存され、次回起動時に復元されます。旧保存データでは既定で展開状態になります。

## フォルダ選択と自動更新

「メディア」オブジェクトを追加し、右側プロパティの「フォルダを選択」を押します。選択したフォルダ内の画像・動画を検出し、ファイル名順またはランダムで再生します。サブフォルダを含める設定にも対応しています。

「自動更新」を有効にすると、表示モード中にメディアプレイヤー単位で更新タイマーが動作します。更新間隔は最低 1 分へ補正され、更新後のファイル一覧は次のファイル切替から反映されます。自動更新に失敗した場合は最後に正常取得できた一覧を維持し、次回更新で再試行します。表示モード終了時やシーン切替時には自動更新タイマーを停止します。編集モードでは「手動更新」でフォルダ名、画像数、動画数、合計件数、最終更新時刻、読込状態を更新できます。手動更新で読込失敗や権限切れが起きた場合も、最後の正常な件数とファイル一覧を維持し、状態欄だけで失敗を知らせます。これらのメディア情報欄と単一画像・動画のファイル名は読取専用です。

## 表示開始と全画面

上部の「▶ 表示開始」を押すと編集UIを隠して表示モードへ切り替わります。表示モードでは 16:9 キャンバスを画面に合わせて等比率表示し、余白は黒背景になります。全画面表示の要求は表示開始時のみ行います。

`Esc` キーで編集モードへ戻ります。ブラウザ側の全画面解除操作で全画面が終了した場合も、`fullscreenchange` を監視して表示モードを停止し、編集モードへ戻ります。停止時は動画を一時停止し、`src` を削除して `load()` を呼び、タイマーと Object URL を破棄します。

## 保存とバックアップ

- 編集内容は IndexedDB へ自動保存されます。
- 「保存」で手動保存できます。
- 「JSON出力」でバックアップを書き出せます。
- 「JSON読込」でバックアップを復元できます。
- JSON 出力には `app: "iSignage"` と `format: "isignage-backup"` の識別情報を含めます。
- PR #1 / PR #2 相当の旧バックアップは、識別情報がなくても `version` と `scenes` の既存構造から判定して読み込めます。
- iSignage形式ではないJSON、空オブジェクト、配列だけのJSON、シーン配列がないJSONは拒否します。
- JSON 読込時は構造を検証し、不正なオブジェクト型や数値は無視または補正します。読込失敗時に現在の編集内容、Undo/Redo履歴、Object URL、IndexedDB保存内容は変更しません。
- JSON にはフォルダアクセス権限、ローカルメディアファイルの実体、単一画像/単一動画の永続的な参照は含まれません。

## ブラウザの権限制約

フォルダ選択は File System Access API の権限に依存します。ブラウザ再起動、端末変更、権限取り消しなどにより、再度フォルダ選択が必要になることがあります。ディレクトリハンドルは IndexedDB に保存しますが、権限自体を JSON へ移行することはできません。コピー、貼り付け、シーン複製したメディアプレイヤーは同じフォルダ参照を共有します。Undo／Redoでフォルダ参照を復元できるよう、オブジェクト削除、シーン削除、フォルダ再選択、JSON読込などの通常操作では使用しなくなったフォルダ参照を直ちには削除しません。新規作成または初期化時に、保存済みのフォルダ参照をまとめて整理します。

## 音声付き動画の自動再生制限

ブラウザは音声付き動画の自動再生を制限する場合があります。そのため動画とメディアプレイヤーの既定値はミュートです。音声付きで再生したい場合は、ブラウザ設定や利用者操作の条件に注意してください。

## 別端末へ移行する場合の注意

JSON バックアップを移行先端末で読み込んだ後、画像・動画ファイルやメディアフォルダは移行先で再指定してください。ローカルファイルそのものやフォルダ権限はバックアップに含まれません。

## 実機確認状況

このリポジトリ更新では、GUI ブラウザが利用できない実行環境のため Edge / Chrome の手動実機確認は未実施です。構文検証を行っています。ヘッドレス初期ロード確認は、この実行環境にブラウザがないため未実施です。実機では、フォルダ選択、全画面解除、動画自動再生、メディア自動更新を重点的に確認してください。

## 既知の制約

- `file://` 実行を前提とするため、ブラウザや企業ポリシーにより File System Access API が制限される場合があります。
- 単一画像・単一動画の File オブジェクトはブラウザの保存仕様に制約があるため、長期運用ではメディアプレイヤーのフォルダ指定を推奨します。
- 回転したオブジェクトの当たり判定やリサイズは簡易実装です。
- 背景画像は v0.1.1 では未対応です。背景色はシーン設定から変更できます。
- 外部通信、CDN、npm、サーバー処理は使用していません。
